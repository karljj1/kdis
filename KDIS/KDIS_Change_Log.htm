KDIS Change Log

Version Numbers:

X -Y -Z

X - Major Change, changing DIS version.
Y - Medium, introduction of new features, data types or PDU etc

Z - Minor, small changes, new functions, comments, bug fix etc.

Please post bugs/suggestions/comments to the SourceForge forum/tracker

  • Added new function to all enumerations - GetEnumAsString<Enum Name>( KINT32 Value ). These functions will return the string value for an enumeration. E.G If you wanted to know what the value 39 represented in the enumeration EmitterFunction you would call GetEnumAsStringEmitterFunction( 39 ) and in this case the returned value would be "ClickJammingFunction". This feature can be enabled/disable by declaring a macro KDIS_USE_ENUM_DESCRIPTORS in the pre-processor definitions or the KDefines.h file. This feature can make your generated exe/dll/lib significantly larger so disable it if you don't plan to use it. When disabled the GetEnumAsString functions will simply return the enumerated value as a string. E.G GetEnumAsStringEmitterFunction( 39 ) would now return "39". I am currently applying this change to all GetAsString function in Data types and PDU. More infomation can be found here. This feature can also be enabled/disabled from CMake.
